# ADR-029：主动感知与多触发机制

> **状态**：✅ 已决策
> **日期**：2026-06-05
> **作者**：竞品分析 → 系统架构组

---

## 背景

竞品分析发现 **G2/G3 差距**：

- G2：Sivan 缺乏主动感知能力（位置、时间、视觉），是纯粹的"被动执行"模式
- G3：触发机制单一，只有手动 + Cron，缺少事件/条件驱动触发（对标 Shortcuts、Alexa Routines）

## 决策

在森林架构中扩展 Trigger 机制，从单一手动触发升级为五类触发器：

| 触发器 | 触发条件 | 使用场景 |
|--------|---------|---------|
| `ManualTrigger` | 用户主动输入 | 对话输入、快捷指令 |
| `CronTrigger` | 定时表达式 | 日报、周报、定时提醒 |
| `EventTrigger` | 外部事件（Webhook、系统通知） | 邮件到达、CI 完成 |
| `LocationTrigger` | 地理围栏 | 到家/离开自动执行 |
| `ConditionTrigger` | 条件表达式（值 > 阈值） | 股票监控、传感器告警 |
| `GoalDependencyTrigger` | 依赖的目标树完成 | 编排链式任务 |

```java
sealed interface Trigger permits ManualTrigger, CronTrigger, EventTrigger,
    LocationTrigger, ConditionTrigger, GoalDependencyTrigger {
    boolean shouldFire(Context ctx);
}
```

## 理由

1. 竞品（Rabbit R1、Shortcuts、Alexa）已验证多触发方式是提升主动性的关键路径。
2. Cron + Event + Location + Condition 覆盖了个人助理的核心场景。
3. 与森林架构中的 ExecutableNode 一致——Trigger 触发 → 创建 GoalTree → 执行。
