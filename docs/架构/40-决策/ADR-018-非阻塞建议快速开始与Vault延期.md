# ADR-018：非阻塞建议——快速创建首个 GoalTree 与 Vault 延期

> **状态**：⏸ 暂缓（实现阶段按需采纳）
> **日期**：2026-06-05
> **作者**：产品经理 & 安全架构师初审 → 系统架构组

---

## 背景

初审（review-第三方评审报告.md）结项时评审人提出两项非阻塞建议：

### 建议 1（王老师，R7 延伸）

引导卡片中的"试试快速开始"按钮，可以改为直接帮用户创建第一个 GoalTree（如"写一个 Hello World"），而不是跳转到空对话。让用户首次操作就有产出感。

### 建议 2（李工，SecretStore 延伸）

密钥管理 Vault 集成可以在上线后按需加入。当前 `SecretStore` 的环境变量默认实现已满足 v2.0 需求。

## 决策

### 建议 1：实现阶段按需采纳

在 Phase 3（交互层开发）时评估：
- 如果 OnboardingGuide 已有"示例对话"功能，可自然引导用户创建首个 GoalTree
- 如果用户体验目标更倾向于"自由探索"，则保持空对话入口更合适
- **不提前锁定 UI 方案**，待交互原型确定后再决定

### 建议 2：Vault 集成延期

- 当前 `SecretStore` 接口已预留扩展点（ADR-010 `AuditedSecretStore`）
- Vault 集成可在外包管理面板开发时（Phase 4）按需实现
- 环境变量 + `AuditedSecretStore` 覆盖 v2.0 所有安全审计需求

## 理由

1. **建议 1** 的取舍取决于产品 UX 设计方向，非架构决策，不应在架构阶段锁定。
2. **建议 2** 的 Vault 集成需要 Vault 基础设施运维，v2.0 阶段引入徒增部署复杂度。
