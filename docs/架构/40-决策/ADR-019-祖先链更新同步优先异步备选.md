# ADR-019：CompressibleNode 祖先链更新策略——同步优先，按需异步

> **状态**：⏸ 暂缓（等待 Phase 0 压测数据）
> **日期**：2026-06-06
> **作者**：交叉评审 → 系统架构组

---

## 背景

交叉评审发现 **刘-R1 关注点**：`CompressibleNode.onStatusChanged()` 从叶子往根遍历更新祖先链，在极端不平衡树（链表状，深度 1000）中一次完成需要更新 1000 个祖先节点。

## 决策

采用「同步优先，按需异步」策略：
- **Phase 0 实现同步版本**：SafeTokenCache 的 `synchronized` 保护串行化链更新
- **Phase 0 压测验证**：压测 1000 节点深度不平衡树的叶子完成延迟
- **如果延迟 > 50ms**：切换为异步惰性失效——叶子仅标记自身 stale，祖先链按需重算

## 理由

1. 不平衡深度 1000 是极端情况，实际 AI 对话树的分支因子通常 > 3，深度 < 20。
2. 同步实现在典型场景下更简单、更可预测。
3. 压测数据可验证是否值得引入异步复杂性。
