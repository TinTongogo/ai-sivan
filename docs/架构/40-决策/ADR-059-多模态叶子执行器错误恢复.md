# ADR-059：多模态叶子执行器错误恢复

> **状态**：✅ 已决策
> **日期**：2026-06-06

## 背景

多模态生成/分析可能因多种原因失败：模型不支持、配额不足、内容安全过滤等。当前没有明确的错误分类和恢复策略。

## 决策

| 错误类型 | 错误码 | 恢复策略 |
|---------|--------|---------|
| 模型不支持 | `MODEL_NOT_SUPPORTED` | 自动降级为文本描述 |
| 配额不足 | `QUOTA_EXCEEDED` | 提示用户检查 API 配额，切换到备用模型 |
| 内容安全过滤 | `CONTENT_FILTERED` | 提示用户修改输入 |
| 网络超时 | `TIMEOUT` | 自动重试 1 次 |
| 未知错误 | `UNKNOWN` | 返回错误详情 + traceId |

每个多模态叶子执行器必须实现至少两级错误恢复（降级 + 用户提示）。

## 理由

用户需要区分失败原因并做后续处理。统一错误分类避免了"生成失败"这样笼统的提示。
