# ADR-057：Flashback 推送频率与用户控制

> **状态**：✅ 已决策
> **日期**：2026-06-06

## 背景

Flashback 通过 WebSocket 主动推送，但未提供场景级别开关和频率限制。无用户控制粒度，可能导致对话刷屏。

## 决策

| 控制维度 | 默认值 |
|---------|--------|
| 每会话次数上限 | 3 次 |
| 频率限制 | 1 条/分钟，5 条/小时 |
| 屏蔽标签 | 用户可配置（如"工作"、"购物"） |
| 推送优先级 | 权重 > 0.8 立即推，0.6~0.8 排队，< 0.6 不推 |

## 理由

Flashback 的核心价值是"刚好在需要时出现"。不加控制的推送会从"贴心"变成"骚扰"。
