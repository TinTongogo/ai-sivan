# ADR-013：前端递归渲染性能验证

> **状态**：✅ 已决策
> **日期**：2026-06-06
> **作者**：第三方终审 → 系统架构组

---

## 背景

第三方终审发现 **C11 关注点**：前端无第三方 UI 库的约束下，ForestTree 递归树组件是最复杂的 UI 元素。1000 节点的 DOM 渲染性能未经验证，开发成本可能超出预估。

架构评审（R06）独立发现了同一问题。

## 决策

在实现前做 1-2 天的原型验证，确认 ForestTree 递归组件的渲染性能：

### 原型验证方案

```html
<!-- 原型测试：mock 1000 节点树，测量首次渲染时长 -->
<ForestTree :nodes="mockNodes(1000)" :depth="0" />
```

### 验收标准

| 指标 | 目标 | 降级方案 |
|------|------|---------|
| 首次渲染（1000 节点） | < 200ms | 虚拟滚动（vue-virtual-scroller） |
| 增量更新（单节点 status 变更） | < 16ms（60fps） | 批量更新合并 |
| 折叠/展开 | < 50ms | 折叠子树不产生 DOM（已有设计） |

### 如果原型不达标

1. **折叠子树不生成 DOM**：当前设计已做（见 §6.2 折叠策略），确保折叠节点不渲染子 DOM。
2. **`content-visibility: auto`**：CSS 属性，浏览器原生跳过可视区外渲染。
3. **虚拟滚动降级**：`vue-virtual-scroller` 或自行实现，仅渲染可视区域节点（预计可实现 10000+ 节点流畅渲染）。

## 理由

1. **降低风险**：无第三方 UI 库的约束下，递归树组件是最高风险项。1-2 天原型验证成本远低于实现后发现性能问题返工的成本。
2. **提供决策数据**：原型数据决定是否启动虚拟滚动降级方案，而非在实现阶段凭感觉选择。
3. **测试框架自然产出**：原型代码可直接演化为 `<ForestTree>` 的单元测试。
