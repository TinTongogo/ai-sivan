# ADR-034：跨聚合根事务边界——TemplateService 与 GoalTreeCompleted

> **状态**：✅ 已决策
> **日期**：2026-06-06

## 背景

`GoalTreeCompleted` 事件被 `TemplateService.updateSuccessRate()` 订阅。该操作更新 `shared_templates` 表，与 `forest_nodes` 表的节点状态更新不在同一事务中。如果模板成功率更新失败，节点已完成但模板统计丢失，造成数据不一致。

## 决策

| 阶段 | 方案 | 说明 |
|------|------|------|
| Phase 0/1 | `@Transactional(propagation = REQUIRES_NEW)` | 模板统计失败不影响节点状态，独立事务 |
| Phase 2+ | Transactional Outbox | 节点状态与领域事件写入同一本地事务，由转发器确保至少投递一次 |

## 理由

1. 节点状态是核心数据，模板统计是辅助数据。核心数据的正确性不应受辅助数据影响。
2. REQUIRES_NEW 在 Phase 0/1 足够简单，引入发件箱模式需要额外表 + 调度器。
