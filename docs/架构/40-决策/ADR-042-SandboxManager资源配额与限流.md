# ADR-042：SandboxManager 资源配额与限流

> **状态**：✅ 已决策
> **日期**：2026-06-06

## 背景

`SandboxManager.execute()` 只做策略校验，不对文件读写、shell 命令的执行时间、内存使用做限制。恶意用户可通过大量并发 `bash` 或大文件写入造成 DoS。

## 决策

| 资源 | 限制 | 超过处理 |
|------|------|---------|
| 文件读取大小 | 10 MB | 拒绝 |
| 文件写入大小 | 5 MB | 拒绝 |
| Shell 执行时间 | 60 秒 | 强制 kill |
| HTTP 请求超时 | 15 秒 | 超时断开 |
| 并发 Action 数 | 20 | 排队等待 |

超时按 Action 类型独立配置。超过并发数时排队等待而非直接拒绝。

## 理由

资源配额是沙箱安全的基本要求——不做限制的沙箱等于没有沙箱。
