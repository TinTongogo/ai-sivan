# ADR-037：深树递归调度器策略——优先 parallel()，谨慎使用 boundedElastic()

> **状态**：✅ 已决策
> **日期**：2026-06-06

## 背景

深度 > 50 时切换到 `Schedulers.boundedElastic()` 有两个已知风险：Reactor Context 在线程切换时可能丢失；boundedElastic 默认最大线程数 = 10 × CPU 核数，100 并发可创建数百个线程。

## 决策

| 调度器 | 适用场景 |
|--------|---------|
| `Schedulers.parallel()` | 深树递归遍历（非阻塞计算），固定线程数 = CPU 核数 |
| `Schedulers.boundedElastic()` | 仅在确实需要阻塞调用（如 LLM 同步请求）时使用 |

切换时显式传递 `Context`：`.contextWrite(ctx)`。

## 理由

parallel() 线程数固定，不会随并发数膨胀；配合 `Hooks.enableAutomaticContextPropagation()` 可保证 Context 传播。
