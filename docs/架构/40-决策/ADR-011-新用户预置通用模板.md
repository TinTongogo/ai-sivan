# ADR-011：新用户预置通用模板

> **状态**：✅ 已决策
> **日期**：2026-06-06
> **作者**：第三方终审 → 系统架构组

---

## 背景

第三方终审发现 **C9 关注点**：新用户开始时没有执行历史，本能模板的 `FeedbackHandler` 和 `ExplorationScheduler` 都在空跑（usageCount = 0）。用户前几次使用大概率走 LLM 生成路径，体验和纯 LLM 方案没有区别。

架构评审（R05）独立发现了同一问题。

## 决策

为首次使用的用户预置一组经过验证的通用模板，内置在应用中（不占模板共享池）：

| 模板 | 用途 | 初始 usageCount | 初始 successRate |
|------|------|-----------------|-----------------|
| 代码审查 | review code diff | 999 | 0.92 |
| 周报生成 | weekly report | 999 | 0.90 |
| 简单对话 | simple chat | 999 | 0.95 |
| 内容总结 | content summarize | 999 | 0.93 |
| 问题分析 | problem analysis | 999 | 0.91 |

### 设计要点

1. **预置模板标记**：`isPreset = true`，区别于用户自定义模板。预置模板不参与 `ExplorationScheduler` 的淘汰和替换。
2. **初始统计值**：`usageCount=999` + `successRate=0.85~0.95`，确保在 FeedbackHandler 的排序中排在前列。
3. **版本更新**：预置模板随应用版本更新，不依赖模板共享池的网络同步。
4. **混合排序**：`FeedbackHandler.sort()` 中预置模板与用户模板按同一公式排序，不特殊优待。

## 理由

1. **提升首次体验**：新用户第一次使用就能命中模板，感受到"系统理解我的需求"，而非"和 ChatGPT 一样"。这是产品的核心差异化。
2. **加速反馈闭环**：预置模板的初始统计数据让 `ExplorationScheduler` 在新用户场景下也能做出有意义的对比。
3. **零维护成本**：模板内置在应用中，不依赖外部服务。后续可通过应用更新增删。
