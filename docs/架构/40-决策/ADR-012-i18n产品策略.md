# ADR-012：i18n 产品策略

> **状态**：✅ 已决策
> **日期**：2026-06-06
> **作者**：第三方终审 → 系统架构组

---

## 背景

第三方终审发现 **C10 关注点**：`00-总览与路线图.md` 的 §3.8 国际化只有 5 条技术约束，没有涉及多语言市场策略、翻译质量保障、日期/数字格式本地化等产品层面的设计。

## 决策

在 `00-总览与路线图.md` 中新增 **§3.9 国际化产品策略**，覆盖以下产品层面：

### 支持语言

| 优先级 | 语言 | 计划阶段 |
|--------|------|---------|
| P0 | 中文（zh-CN） | Phase 0（默认语言） |
| P0 | 英文（en） | Phase 0 |
| P1 | 日文（ja）、韩文（ko） | Phase 3 |
| P2 | 阿拉伯文（ar，RTL）、泰文（th） | Phase 4 |

### LTR/RTL 布局适配

- 前端使用 CSS 逻辑属性（`margin-inline-start` 替代 `margin-left`）而非物理属性
- 阿拉伯文等 RTL 语言自动切换 `dir="rtl"`
- 组件级无 LTR/RTL 硬编码

### 翻译管理

- 所有 i18n key 集中管理在 `sivan-i18n/` 模块，前端和后端共享 key 命名空间
- 接入第三方翻译管理平台（暂定 Crowdin / Lokalise）
- 翻译质量审核流程：机翻初稿 → 人工审核 → 社区反馈 → 定期更新

### 日期/数字/货币格式

- 使用 `java.time.format.DateTimeFormatter` 本地化格式
- 数字格式化遵循 `java.text.NumberFormat` 的 locale 感知

## 理由

1. **产品差异化**：Sivan 的目标市场包括海外用户，技术层面的 i18n 支持不足以覆盖产品体验。
2. **渐进实施**：P0 仅中文+英文，不阻塞 Phase 0 开发。P1/P2 语言在 Phase 3/4 按需追加。
3. **统一管理**：前后端共享 i18n key 命名空间，避免同一文案在两套系统中定义两次。
4. **翻译质量**：接入专业翻译管理平台，而非依赖开发者自行翻译。
