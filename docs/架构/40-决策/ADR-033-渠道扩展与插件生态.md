# ADR-033：渠道扩展与插件生态

> **状态**：📋 已识别（Phase 3-4 规划）
> **日期**：2026-06-05
> **作者**：竞品深度分析 → 系统架构组

---

## 背景

竞品深度分析发现三项战略性差距：

| 差距 | 描述 | 影响 |
|------|------|------|
| Deep-Gap-1 | 渠道覆盖太窄（仅 Web） | 优先级：Telegram/Discord/Slack/WeCom (P1), Watch/Mobile (P2) |
| Deep-Gap-2 | 插件生态太小 | 短期：`skill2mcp` 适配器接入 ClawHub 13K+ skills；中期：`ClawHubDiscoverer`；长期：skill 到 ToolProvider 升级路径 |
| Deep-Gap-3 | 部署体验缺失 | Docker Compose 一键启动脚本（约 1 天） |

## 决策

### Deep-Gap-1：渠道扩展

| 优先级 | 渠道 | 计划阶段 |
|--------|------|---------|
| P1 | Telegram / Discord / Slack / 企业微信 | Phase 3 |
| P2 | Watch / Mobile (iOS/Android) | Phase 4 |

渠道适配器接口与 `EventSink` / `Delivery` 一致——每个渠道实现一个 `ChannelAdapter`，将 `OrchestrationEvent` 转换为渠道原生格式。

### Deep-Gap-2：插件生态扩展

三个阶段：

1. **短期**：开发 `skill2mcp` 适配器，将 ClawHub 社区的 13K+ 技能自动映射为 MCP 工具
2. **中期**：`ClawHubDiscoverer` 作为 ToolProviderManager 的扩展，从 ClawHub 发现技能
3. **长期**：提供 skill → ToolProvider 升级路径，技能发布者可将作品迁移到 ToolProvider 接口

### Deep-Gap-3：部署体验

Phase 0 交付时附带 `docker-compose.yml`（含 PostgreSQL + 应用 + Ollama），实现一键启动。预估 1 天。

## 理由

1. 渠道扩展是触达更广泛用户的基础设施——当前仅 Web 覆盖了不到 50% 的目标用户。
2. 插件生态复用 ClawHub 现有技能，无需从零构建市场。
3. Docker Compose 是开发者和早期用户的最低部署门槛。
