# ADR-056：首次使用引导状态后端持久化

> **状态**：✅ 已决策
> **日期**：2026-06-06

## 背景

`OnboardingGuide` 使用 `localStorage` 记录 `onboarding_done`。用户更换设备或清除缓存后会再次显示引导。

## 决策

引导完成状态写入后端 `user_profiles` 表：

```sql
ALTER TABLE user_profiles ADD COLUMN onboarding_done BOOLEAN DEFAULT false;
```

前端首次加载调用 `GET /api/v2/user/profile` 获取状态，完成后调用 `PUT /api/v2/user/profile` 持久化。

## 理由

跨设备同步是基本的产品体验——用户不应在每台设备上都被迫重新看完引导。localStorage 仅在无网络时做降级缓存。
