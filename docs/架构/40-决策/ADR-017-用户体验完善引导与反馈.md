# ADR-017：用户体验完善——引导、异常反馈与模板感知

> **状态**：✅ 已决策
> **日期**：2026-06-05
> **作者**：产品经理初审 → 系统架构组

---

## 背景

初审（review-第三方评审报告.md）中产品经理王老师发现三项风险：

**R7：用户引导缺位。** 新用户首次打开应用时面对空白界面，不知道能做什么，没有首次使用引导流程。

**R8：异常反馈不足。** 系统执行失败或出错的反馈信息太笼统，用户无法理解问题原因和下一步操作。

**R9：模板匹配过程对用户不可见。** 系统自动匹配了本能模板进行执行，但用户完全不知道"系统正在用哪个模板执行我的请求"，匹配过程对用户透明。

## 决策

### R7：首次使用引导 `OnboardingGuide.vue`

新增首次使用引导组件：
- 首次登录时展示引导卡片，介绍核心功能（GoalTree、模板、Flashback）
- "试试快速开始"按钮进入示例对话
- 模板探索入口：展示预置模板列表供用户浏览
- 首次展开提示：用户首次展开 GoalTree 时增加动画说明

### R8：三级错误反馈

```
错误反馈结构：
┌─ 对话层：用户可读的错误描述"联网失败，已切换到离线模式"
├─ 详情层：可操作建议"检查代理配置或重试"
└─ 技术日志层：traceId + Duration + 异常栈（可复制）
```

- SSE 事件中携带错误层级标识
- 对话层默认展示，详情层点击展开，技术日志层开发者模式可见
- 所有错误携带 `traceId`，便于反馈到技术支持

### R9：模板匹配感知

- SSE 首条事件携带模板匹配信息（`"matchedTemplate": "code-review"`）
- 执行完成后摘要显示模板 usageCount + successRate
- 用户可展开查看匹配理由："匹配到代码审查模板（置信度 87%）"

## 理由

1. **R7**：AI 产品的首次体验决定了次日留存率，空白界面 = 零引导 = 用户流失。
2. **R8**：三级错误反馈覆盖了从普通用户到开发者所有角色的需求。普通用户不需要看到异常栈，但开发者调试时需要。
3. **R9**：模板匹配是系统的核心差异化能力，用户需要感知"系统在帮我做匹配"才能建立信任。"做完后展示使用率和成功率"可自然转化为产品粘性指标。
