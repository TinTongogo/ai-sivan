# ADR-049：SSE 断线重连与进度续传

> **状态**：✅ 已决策
> **日期**：2026-06-06

## 背景

SSE 连接中断后，前端无法得知当前 GoalTree 执行进度，只能重新开始执行。无 `Last-Event-ID` 或 `resume_token`。

## 决策

| 机制 | 说明 |
|------|------|
| 快照端点 | `GET /api/v2/goals/{forestId}/progress`，返回当前节点状态列表 |
| 重连顺序 | 前端先获取快照恢复 UI → 再订阅 SSE 流 |
| Last-Event-ID | SSE 标准字段，后端据此决定事件回放起始点 |

## 理由

断线重连是持久连接的基本要求。快照端点 + SSE 续传的组合覆盖了全部断线场景。
