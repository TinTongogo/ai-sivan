# ADR-051：i18n 资源加载策略

> **状态**：✅ 已决策
> **日期**：2026-06-06

## 背景

要求用户可见文本走 i18n，但未说明是前端静态资源还是后端动态返回。SSE 事件中的 `message` 字段如何根据 `Accept-Language` 动态选择未设计。

## 决策

| 资源类型 | 加载方式 |
|---------|---------|
| 前端 UI 文本 | 前端静态 JSON 文件（构建时打包） |
| 后端错误消息 | 后端 i18n 资源文件，SSE 事件 `message` 已本地化 |
| LLM 输出 | 不翻译，按用户输入语言回复 |

SSE 事件中的 `message` 在后端按 `Accept-Language` Header 渲染后发送，前端不承担运行时翻译责任。

## 理由

前端只渲染已本地化的文本，不在前端做运行时翻译，避免翻译逻辑重复和文案不一致。
