# ADR-038：metadata 内存预算与约束

> **状态**：✅ 已决策
> **日期**：2026-06-06

## 背景

10000 节点 < 500MB 的估算未包含 `metadata`（`Map<String, Object>`）开销。每个节点存储 5 个键值对时，10000 节点额外 4-50 MB（取决于 value 大小）。

## 决策

| 约束 | 值 |
|------|-----|
| 键值对上限 | 10 个/节点 |
| 单 value 上限 | 1 KB |
| 超限处理 | 架构评审 |

大数据（完整文档、Base64 图片）存储为独立实体，metadata 中仅保留引用 key。

## 理由

典型场景下 metadata 开销在可接受范围（10000 节点 < 50 MB），但应避免在 metadata 中存储大数据对象。
