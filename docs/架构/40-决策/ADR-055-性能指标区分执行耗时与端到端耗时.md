# ADR-055：性能指标区分执行耗时与端到端耗时

> **状态**：✅ 已决策
> **日期**：2026-06-06

## 背景

`goal.tree.duration` 记录了从 `execute` 到 `complete` 的耗时。对于 SUMMARY 模式，用户可能数小时后才查询结果，该时长会误导性能分析。

## 决策

将 `goal.tree.duration` 拆分为两个指标：

| 指标 | 含义 |
|------|------|
| `goal.tree.execution_duration` | 实际执行耗时，从 execute() 到最后一个节点完成 |
| `goal.tree.total_duration` | 端到端耗时，从创建 GoalTree 到用户首次读取结果 |

同时按 `p50/p95/p99` 分位值和 `STREAM/SUMMARY` 场景区分。

## 理由

SUMMARY 模式下挂起时间不应计入执行性能分析。
