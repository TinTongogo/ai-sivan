# ADR-032：MCP 零预置 + 按需发现设计原则

> **状态**：✅ 已决策
> **日期**：2026-06-05
> **作者**：竞品分析 → 系统架构组

---

## 背景

竞品分析发现 **G9 差距**：对比 Dify/Coze 提供了丰富的预置 MCP 集成（Gmail、Slack、GitHub 等），Sivan 的 MCP 集成需要用户自行搭建和配置，非技术用户的使用门槛高。

## 决策

坚持「零预置 + 按需发现」设计原则，不预置批量 MCP 集成：

```
Coze / Dify（预置市场）：
  用户 → 打开市场 → 浏览 100+ 插件 → 选择 → 安装 → 授权
  → 需要主动管理"插件库"，得先知道有什么、该装什么

Sivan（按需发现）：
  用户 → "帮我审查这个 PR"
  系统 → "需要连接 GitHub 来获取 PR 信息，要配置吗？"
  用户 → "好" → OAuth 授权 → 执行
  → 不需要知道"插件"这个概念
  → 系统在需要时引导，不需要时不出现
```

## 理由

1. **枢纽原则**：不给用户塞不需要的东西，在需要时引导配置。
2. **认知负担降低**：非技术用户面对一堆预置集成会迷茫，按需发现只在需要时出现一次。
3. **零维护成本**：不需要维护集成市场、不需要处理版本兼容性、不需要审核第三方集成。
4. **开放协议优势**：MCP 是开放协议，用户可自行搭建任何 MCP 服务器。
