# ADR-054：事件持久化与事务性发件箱

> **状态**：✅ 已决策（分阶段实施）
> **日期**：2026-06-06

## 背景

`GoalTreeCompleted` 的订阅者若执行失败（DB 临时不可用），事件丢失且无重试。虽然有 `@Retryable`，但默认同步事件会回滚节点状态。

## 决策

| 阶段 | 方案 |
|------|------|
| Phase 0/1 | 同步事件 + `@Transactional(REQUIRES_NEW)`，订阅者失败不影响主事务 |
| Phase 2+ | Transactional Outbox：节点状态与事件写入同一事务，`OutboxForwarder` 轮询投递 |

Outbox 表字段：`event_id`, `event_type`, `payload`, `status`, `retry_count`, `created_at`。投递成功标记为 `processed`，失败重试最多 3 次（指数退避），超过后进入死信队列。

## 理由

同步事件在 Phase 0/1 足够简单。Phase 2 引入发件箱模式，确保事件至少投递一次。
