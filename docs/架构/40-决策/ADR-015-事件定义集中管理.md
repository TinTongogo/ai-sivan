# ADR-015：事件定义集中管理

> **状态**：✅ 已决策
> **日期**：2026-06-05
> **作者**：首席架构师初审 → 系统架构组

---

## 背景

初审发现 **R3 关注点**：事件定义分散在多个设计文档中（01、05、08 等），不同文档中同类事件签名可能不一致，导致实现阶段出现"事件对不上"的问题。

## 决策

所有领域事件集中定义在 `00-共享事件.md` 中，作为唯一来源。

```java
// 00-共享事件.md 集中定义
public record GoalTreeCompleted(String forestId, String accountId) {}
public record MilestoneCompleted(String forestId, String nodeId, String accountId) {}
public record NodeStatusChanged(String forestId, String nodeId, NodeStatus newStatus, String accountId) {}
public record NodeExecutionFailed(String forestId, String nodeId, String reason, String accountId) {}
```

各设计文档只引用 `00-共享事件.md` 中的事件，不自行定义。

### 规则

- 新增事件必须定义在 `00-共享事件.md`，不在各设计文档中独立定义
- 事件必须携带 `forestId` + `accountId`，确保跨上下文可追溯
- 事件使用 Java `record` 类型，天然不可变、带 equals/hashCode

## 理由

1. **单一事实来源**：不存在"某文档更新了事件但其他文档没同步"的问题。
2. **编译器检查**：所有模块引用同一组 Java record，签名不一致（字段名、类型不匹配）直接编译报错。
3. **文档即代码**：`00-共享事件.md` 就是事件注册表，新增事件时一目了然。
