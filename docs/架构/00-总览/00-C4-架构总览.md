# C4 架构总览

> 日期：2026-06-05
> 状态：设计草案

本文档使用 C4 模型（Context → Container → Component → Code）描述 Sivan v2.0 的整体架构。
关键交互节点附带时序图和流程图。

---

## L1 — Context（系统上下文）

```mermaid
C4Context
  title Sivan v2.0 系统上下文

  Person(user, "用户", "通过对话/语音使用私人 AI 助理")

  System(sivan, "Sivan v2.0", "私人 AI 助理\n树模型编排 + 多模型路由 + 工具执行")

  System_Ext(llm, "LLM Providers", "OpenAI / Anthropic / 本地 vLLM\n文本对话 + 图像生成 + TTS")
  System_Ext(mcp, "MCP Servers", "智能家居 / 文件系统 / 浏览器\n通过工具协议接入")
  System_Ext(db, "PostgreSQL", "森林节点存储\n对话历史 / 模板 / 记忆")
  System_Ext(tts, "TTS Engine", "系统语音输出")
  System_Ext(idp, "OAuth Provider", "Google / GitHub\nOAuth 2.1 + PKCE 登录")

  Rel(user, sivan, "对话、指令、语音")
  Rel(user, sivan, "注册、登录、管理项目")
  Rel(sivan, llm, "模型调用")
  Rel(sivan, mcp, "工具执行")
  Rel(sivan, db, "读写")
  Rel(sivan, tts, "语音合成")
  Rel(sivan, idp, "OAuth 登录")

  UpdateLayoutConfig($c4ShapeInRow="3", $c4BoundaryInRow="1")
```

**说明**：
- 用户通过对话或语音与 Sivan 2.0 交互
- 用户需要先注册/登录（邮箱密码或 OAuth）才能使用系统
- Sivan 2.0 内部以树模型编排执行，不区分 CHAT / SINGLE_AGENT / SQUAD——所有输入都是树
- 所有请求携带 JWT 标识 accountId，贯穿全链路（零 ThreadLocal）
- LLM Providers 支持多模型（文本 + 图像 + 语音），通过统一的 `LanguageModel` 接口调用
- MCP Servers 提供工具执行能力（文件操作、设备控制等）
- PostgreSQL 是唯一的持久化存储，核心表为 `forest_nodes`（递归树结构）
- OAuth Provider 提供第三方认证登录（OAuth 2.1 + PKCE）

## L2 — Container（容器视图）

```mermaid
C4Container
  title Sivan v2.0 容器架构

  Person(user, "用户")

  System_Ext(llm, "LLM Providers")
  System_Ext(mcp, "MCP Servers")
  System_Ext(db, "PostgreSQL")

  Boundary(b1, "Sivan v2.0 Backend") {
    Container(api, "API Gateway", "Spring Boot WebFlux", "REST + SSE + WebSocket 端点\n认证过滤器 + CORS")
    Container(acct, "Account Service", "Java 21", "注册/登录/OAuth\nJWT 签发/校验\n账号管理")
    Container(proj, "Project Service", "Java 21", "项目 CRUD\n归档/恢复/删除\n目录初始化")
    Container(forest, "Forest Executor", "Java 21 + Reactor", "递归树执行引擎\n五种 mode 调度\n叶子执行器派发")
    Container(router, "Model Router", "Java 21", "能力查询 + 模型选择\n回退 + 负载均衡")
    Container(compressor, "Forest Compressor", "Java 21", "递归折叠 + 策略分发\n上下文注入")
    Container(matcher, "Tree Matcher", "Java 21", "本能模板匹配\nEmbedding 语义匹配\nLLM 生成")
    Container(memory, "Memory Service", "Java 21", "记忆存储/检索/衰减\nFlashback 注入")
    Container(sandbox, "Sandbox", "Java 21", "进程隔离 + 文件沙箱\n权限校验 + 审计日志")
    Container(eventbus, "Event Bus", "Spring ApplicationEventPublisher", "领域事件发布/订阅")
  }

  Rel(user, acct, "注册/登录")
  Rel(user, proj, "管理项目")
  Rel(user, api, "对话/指令/语音（携带 JWT）")
  Rel(api, acct, "认证过滤器委托")
  Rel(api, proj, "查询项目目录")
  Rel(api, forest, "委托执行树")
  Rel(forest, router, "获取语言模型")
  Rel(forest, sandbox, "工具执行前授权")
  Rel(forest, eventbus, "发布 NodeStatusChanged")
  Rel(router, llm, "统一接口调用")
  Rel(forest, mcp, "工具调用")
  Rel(api, compressor, "压缩上下文注入 LLM")
  Rel(compressor, memory, "获取记忆")
  Rel(compressor, db, "读取节点")
  Rel(api, matcher, "任务 → 树模板")
  Rel(api, db, "CRUD")
```

### 容器职责

| 容器 | 入口 | 核心接口 | 关键依赖 |
|---|---|---|---|
| API Gateway | HTTP / WebSocket | 认证过滤器 + `ForestTree.vue` 前端 | Account Service, Forest Executor |
| Account Service | `POST /auth/register` | `AuthService.login()` / `JwtTokenProvider` | OAuth Provider, DB |
| Project Service | `POST /projects` | `ProjectService.create()` / `ShortIdGenerator` | DB, FileSystem |
| Forest Executor | `ForestExecutor.execute()` | `ModeStrategy` / `LeafExecutor` | Model Router, Sandbox |
| Model Router | `ModelRouter.forTask()` | `LanguageModel.chat()` | LLM Providers |
| Forest Compressor | `ForestCompressor.compress()` | `FoldStrategy` / `GroupStrategy` | Memory Service |
| Tree Matcher | `TreeMatcher.match()` | `TemplateMatcher` | LLM, DB |
| Memory Service | `FlashbackService.inject()` | `RecallStrategy` | DB |
| Sandbox | `Sandbox.execute()` | `SandboxPolicy` | OS |
| Event Bus | `ApplicationEventPublisher` | `@EventListener` | — |

## L4 — Code（关键交互的时序图）

### 4.1 GoalTree 执行全流程

```mermaid
sequenceDiagram
    participant Client as 前端/调用方
    participant FE as ForestExecutor
    participant MD as ModeDispatcher
    participant LE as LeafExecutor
    participant MR as ModelRouter
    participant LM as LanguageModel
    participant ES as EventSink
    participant DB as PostgreSQL

    Client->>FE: execute(root, ctx)
    activate FE

    Note over FE: root 是一个 InnerGoal(SEQUENTIAL)
    FE->>ES: emit(LIFECYCLE, NODE_START)
    FE->>MD: dispatch(node, ctx, depth)

    activate MD
    MD->>MD: lookup(SEQUENTIAL) → SequentialStrategy
    MD->>FE: execute(node, ctx, depth, next)
    deactivate MD

    Note over FE: executeNode(child[0]) — 第一个子节点
    FE->>ES: emit(LIFECYCLE, NODE_START)
    FE->>LE: executeLeaf(child[0], ctx, sink)

    activate LE
    LE->>MR: forTask(taskProfile)

    activate MR
    MR->>MR: selectBest(taskProfile) → "gpt-4o"
    MR->>LE: LanguageModel (decorated)
    deactivate MR

    LE->>LM: chat(messages, params)

    activate LM
    LM-->>LE: Flux<ChatEvent>
    Note over LM: ChatEvent.Chunk → ChatEvent.Completed
    deactivate LM

    LE-->>FE: OrchestrationEvent
    deactivate LE

    FE->>FE: node.setStatus(COMPLETED)
    FE->>DB: saveNode(node)
    FE->>ES: emit(LIFECYCLE, NODE_END)
    FE-->>Client: SSE event

    Note over FE: 递归下一个子节点...

    FE->>FE: 所有子节点完成
    FE->>ES: emit(LIFECYCLE, NODE_END)
    FE-->>Client: final event (goalId, type, content)
    deactivate FE
```

### 4.2 模型路由与回退

```mermaid
sequenceDiagram

    participant MR as ModelRouter
    participant REG as ModelRegistry

    P-->>LE: Error (503 Service Unavailable)

    LE->>P: chat(messages) (retry 1)

    LE->>F: chat(messages)
    activate F

    LE->>CT: record(model, usage, accountId)

    deactivate CT
```

### 4.5 CONDITIONAL 模式执行

> 五种编排模式的完整流程图见 [01-森林架构-编排与执行.md §3.2](10-设计/01-森林架构-编排与执行.md)（Mermaid 渲染）。

```mermaid
flowchart TB
    START["executeNode(CONDITIONAL)"] --> CHECK["findNextReady(phases, ctx)"]
    CHECK --> FOUND{"有就绪阶段？"}

    FOUND -- "是" --> DISPATCH["phaseExecutor.dispatchPhase(phase)"]
    DISPATCH --> RESULT{"执行结果"}

    RESULT -- "暂停" --> PAUSE["callbacks.onPhasePaused()"]
    RESULT -- "成功" --> COMPLETE["callbacks.onPhaseCompleted()"]

    COMPLETE --> LLM["decideNextPhase(phases, currentIndex, output)"]
    LLM --> DECISION{"LLM 决策"}

    DECISION -- "返回 -1（完成）" --> DONE["return PhaseResult.success()"]
    DECISION -- "返回 N（跳转到阶段 N）" --> CHECK
    DECISION -- "返回 current+1（正常推进）" --> CHECK

    FOUND -- "否" --> ALLDONE{"全部阶段完成？"}
    ALLDONE -- "是" --> DONE
    ALLDONE -- "否" --> CHECKDEAD["检查 dependsOn 环形依赖"]
    CHECKDEAD --> DONE
```
