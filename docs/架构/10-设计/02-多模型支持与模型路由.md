# 多模型支持与模型路由

> 作者：姚永超
> 日期：2026-06-11
> 状态：实施完成（v2 接口层已实现，运行时仍走 v1 Model 路径。AgentLeafExecutor 使用 DefaultModelRouter → v1 Model，未切换至 v2 LanguageModel）

---

## 1. 问题

当前 v1.0 的模型层有三个问题：

1. **`ModelRouter` 混合了路由、能力推断、provider 配置**——职责不单一，扩展一个场景要改多个地方
2. **`ModelCapabilityRegistry` 靠模型名前缀硬编码匹配**——`gpt-4o` → vision+tool+gpt-4 的推断逻辑在字符串 switch 里，新增模型要改代码
3. **`LlmProviderService` 和 Provider 配置耦合在 Spring Bean 中**——无法运行时动态切换或热加载

目标：

- 统一接口：无论 OpenAI / Anthropic / 本地 vLLM，调用方看到的是同一个 `LanguageModel` 接口
- 抹平差异：是否支持 stream、thinking、tools、vision，由模型声明，调用方无需感知
- 动态路由：根据任务类型、成本预算、可用性自动选择最优模型
- 可插拔：新增一个模型提供商 = 实现一个 Adapter + 配一个 yaml

## 2. 核心抽象

### 2.1 LanguageModel — 统一调用接口

所有模型的统一入口。

```java
/**
 * 语言模型——v2.0 中唯一的 LLM 调用接口。
 * 无论底层是 OpenAI、Anthropic 还是本地模型，调用方看到的都是这个接口。
 */
interface LanguageModel {

    /** 模型标识。全局唯一，如 "gpt-4o"、"claude-sonnet-4"、"qwen3-72b"。 */
    String modelId();

    /** 模型能力声明。描述支持哪些特性、不支持哪些。 */
    ModelCapabilities capabilities();

    /**
     * 流式对话接口。
     * 所有特性（stream / thinking / tool_use / vision）通过同一个 Flux 表达：
     *
     * Flux<ChatEvent>:
     *   Chunk(text)        ← 正常 token
     *   Thinking(text)     ← 推理过程（仅 reasoning 模型）
     *   ToolCall(name,args)← 模型请求调用工具
     *   ToolResult(id)     ← 工具调用结果交回后模型继续
     *   Completed(usage)   ← 本次调用结束，含 token 用量
     *   Error(cause)       ← 调用异常
     */
    Flux<ChatEvent> chat(List<Msg> messages, ModelParams params);

    /**
     * 非流式快捷方法。默认通过 Flux.reduce() 实现，
     * 但某些 provider（如 OpenAI）的非流式接口更高效（无需分配 Flux 内部队列）。
     * 各适配器应覆写此方法，优先调用 provider 的原生非流式 API。
     */
    default Mono<ChatResult> complete(ChatRequest request) {
        return chat(request)
            .reduce(new ChatResult(), (acc, event) -> acc.merge(event));
    }
}

> **优化说明**：调用方如果不需要流式输出（如 SUMMARY 模式、模板匹配决策），应优先调用 `complete()` 而非 `chat()`。Provider 适配器应覆写 `complete()` 以调用 provider 的原生非流式接口（如 OpenAI 的 `/v1/chat/completions` 非 streaming 模式），避免不必要的 Flux 开销。

/**
 * 模型能力声明——由 ProviderAdapter 在初始化时填充，不可变。
 * 调用方通过此对象判断"这个模型能不能做 X"。
 */
record ModelCapabilities(
    String modelId,
    String provider,                    // "openai" / "anthropic" / "vllm" / ...
    Set<Capability> supported,
    int maxTokens,
    int maxThinkingTokens,              // 0 表示不支持 thinking
    int maxInput,                       // 上下文窗口
    List<String> supportedToolSchemas,  // tool_use 的 JSON Schema 版本
    double inputPricePer1k,             // 成本信息（美元）
    double outputPricePer1k
) {
    enum Capability { STREAM, THINKING, TOOL_USE, VISION, SYSTEM_PROMPT, JSON_MODE, BATCH }
}
```

### 2.3 ChatEvent / ChatResult — 统一事件协议

```java
sealed interface ChatEvent {
    /** IO 已开始（可用于计算 TTFB）。 */
    record Started() implements ChatEvent {}
    /** 正常 token 输出。 */
    record Chunk(String text) implements ChatEvent {}
    /** 推理过程 token（仅 reasoning 模型）。 */
    record Thinking(String text) implements ChatEvent {}
    /** 模型请求调用工具。 */
    record ToolCall(String id, String name, String args) implements ChatEvent {}
    /** 工具调用结果已交回，模型继续生成。 */
    record ToolResult(String id) implements ChatEvent {}
    /** 调用结束。 */
    record Completed(ChatResult result) implements ChatEvent {}
    /** 调用异常。 */
    record Error(Throwable cause) implements ChatEvent {}
}

record ChatResult(
    String text,
    String thinking,
    List<ChatEvent.ToolCall> toolCalls,
    TokenUsage usage,
    String modelId,
    long durationMs
) {}

record TokenUsage(int inputTokens, int outputTokens, int thinkingTokens, int totalTokens) {
    static TokenUsage ZERO = new TokenUsage(0, 0, 0, 0);
}
```

## 3. Provider Adapter 层

每个模型提供商实现一个 Adapter，将 Provider 的私有 API 转换为统一的 `LanguageModel` 接口。

```
                    ┌──────────────┐
                    │ LanguageModel │  ← 领域层接口
                    └──────┬───────┘
                           │ 实现
         ┌─────────────────┼─────────────────┐
         │                 │                 │
  ┌──────┴──────┐  ┌──────┴──────┐  ┌──────┴──────┐
  │ OpenAiAdapter │  │ ClaudeAdapter│  │ OllamaAdapter│ ← 基础设施层
  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘
         │                 │                 │
  OpenAI API          Anthropic API      Ollama API
```

```java
/**
 * Provider Adapter 基类——定义了从 LanguageModel 到 provider API 的转换骨架。
 */
abstract class ModelProviderAdapter implements LanguageModel {

    /** 子类实现：发出 HTTP 请求，返回 provider 原生的响应流。 */
    protected abstract Flux<ProviderNativeEvent> callProvider(List<Msg> messages, ModelParams params);

    /** 子类实现：将 provider 原生事件转换为统一的 ChatEvent。 */
    protected abstract ChatEvent toChatEvent(ProviderNativeEvent nativeEvent);

    /** 子类实现：将 provider 原生响应解析为 ChatResult。 */
    protected abstract ChatResult toChatResult(ProviderNativeResponse response);

    /** 子类实现：声明本模型的能力。 */
    protected abstract ModelCapabilities declareCapabilities();

    @Override
    public Flux<ChatEvent> chat(List<Msg> messages, ModelParams params) {
        return Flux.concat(
            Mono.just(new ChatEvent.Started()),
            callProvider(messages, params)
                .map(this::toChatEvent)
                .doOnComplete(() -> {
                    if (this.metricsSink != null) {
                        this.metricsSink.emit(MetricEvent.invocation(this.modelId(), 1));
                    }
                })
        );
    }
}
```

### 3.1 OpenAI Adapter 示例

```java
class OpenAiAdapter extends ModelProviderAdapter {
    private final String apiKey;
    private final String modelName;
    private final String baseUrl;
    private final WebClient client;
    private final ModelCapabilities capabilities;

    OpenAiAdapter(OpenAiConfig config) {
        this.apiKey = config.apiKey();
        this.modelName = config.model();
        this.baseUrl = config.baseUrl();
        this.client = WebClient.create(baseUrl);
        this.capabilities = OpenAiCapabilities.infer(modelName);
    }

    @Override
    public String modelId() { return modelName; }

    @Override
    public ModelCapabilities capabilities() { return capabilities; }

    @Override
    protected Flux<ProviderNativeEvent> callProvider(List<Msg> messages, ModelParams params) {
        // 将 Msg 转换为 OpenAI 的 message 格式
        // 发送 POST /v1/chat/completions 流式请求
        // 返回 SSE 事件流
        return client.post()
            .uri("/v1/chat/completions")
            .header("Authorization", "Bearer " + apiKey)
            .bodyValue(buildRequest(messages, params))
            .retrieve()
            .bodyToFlux(OpenAiSseEvent.class);
    }

    @Override
    protected ChatEvent toChatEvent(ProviderNativeEvent nativeEvent) {
        return switch (nativeEvent) {
            case OpenAiSseEvent.Delta d -> new ChatEvent.Chunk(d.text());
            case OpenAiSseEvent.ToolCall t -> new ChatEvent.ToolCall(t.id(), t.name(), t.args());
            case OpenAiSseEvent.Done done -> new ChatEvent.Completed(toChatResult(done));
        };
    }

    @Override
    protected ChatResult toChatResult(ProviderNativeResponse response) {
        return new ChatResult(
            response.text(), null,
            response.toolCalls(),
            new TokenUsage(response.inputTokens(), response.outputTokens(), 0, response.totalTokens()),
            modelName,
            0 // 由调用方填充
        );
    }

    @Override
    protected ModelCapabilities declareCapabilities() {
        return capabilities;
    }
}
```

### 3.2 Adapter 能力推断策略

供应商的能力声明通过 **Strategy 模式**实现，而非硬编码的 if-else 或字符串匹配。

```java
// ———— 能力推断策略接口 ————

interface CapabilityInferrer {
    /**
     * 根据模型名称推断能力。
     * 返回 null 表示本策略不适用于该模型名。
     */
    ModelCapabilities infer(String modelName);
}

// ———— OpenAI 能力的策略实现 ————

class OpenAiCapabilityInferrer implements CapabilityInferrer {

    // 已知模型的精确能力表
    private static final Map<String, ModelCapabilities> KNOWN = Map.of(
        "gpt-4o", new ModelCapabilities("gpt-4o", "openai", Set.of(STREAM, TOOL_USE, VISION, SYSTEM_PROMPT), 4096, 0, 128000, List.of("function"), 2.5e-6, 1.0e-5),
        "gpt-4o-mini", new ModelCapabilities("gpt-4o-mini", "openai", Set.of(STREAM, TOOL_USE, VISION, SYSTEM_PROMPT), 4096, 0, 128000, List.of("function"), 1.5e-7, 6.0e-7),
        "o3-mini", new ModelCapabilities("o3-mini", "openai", Set.of(STREAM, TOOL_USE, SYSTEM_PROMPT), 8192, 0, 200000, List.of("function"), 1.1e-6, 4.4e-6)
    );

    @Override
    public ModelCapabilities infer(String modelName) {
        // 精确匹配
        ModelCapabilities known = KNOWN.get(modelName);
        if (known != null) return known;

        // 前缀匹配：gpt-4o-* 系列继承 gpt-4o 能力
        if (modelName.startsWith("gpt-4o")) return KNOWN.get("gpt-4o").withModelId(modelName);
        if (modelName.startsWith("o3")) return KNOWN.get("o3-mini").withModelId(modelName);

        // 未知模型：提供安全的默认值（最保守）
        return new ModelCapabilities(modelName, "openai", Set.of(STREAM, SYSTEM_PROMPT), 2048, 0, 8192, List.of(), 0, 0);
    }
}

// ———— Anthropic 能力的策略实现 ————

class AnthropicCapabilityInferrer implements CapabilityInferrer {
    @Override
    public ModelCapabilities infer(String modelName) {
        if (modelName.startsWith("claude-sonnet-4")) {
            return new ModelCapabilities(modelName, "anthropic", Set.of(STREAM, THINKING, TOOL_USE, SYSTEM_PROMPT), 8192, 16000, 200000, List.of("function"), 3.0e-6, 1.5e-5);
        }
        if (modelName.startsWith("claude-opus-4")) {
            return new ModelCapabilities(modelName, "anthropic", Set.of(STREAM, THINKING, TOOL_USE, SYSTEM_PROMPT), 8192, 32000, 200000, List.of("function"), 1.5e-5, 7.5e-5);
        }
        // 回退
        return new ModelCapabilities(modelName, "anthropic", Set.of(STREAM, SYSTEM_PROMPT), 2048, 0, 100000, List.of(), 0, 0);
    }
}

// ———— 本地模型（Ollama / vLLM）能力的策略实现 ————

class LocalModelCapabilityInferrer implements CapabilityInferrer {
    @Override
    public ModelCapabilities infer(String modelName) {
        // 本地模型统一按最简能力注册
        return new ModelCapabilities(modelName, "local", Set.of(STREAM, SYSTEM_PROMPT), 2048, 0, 32768, List.of(), 0, 0);
    }
}
```

## 4. ModelRegistry

运行时能力查询 + Provider Adapter 管理。

```java
@Component
class ModelRegistry {
    private final Map<String, LanguageModel> models = new ConcurrentHashMap<>();
    private final List<CapabilityInferrer> inferrers;

    ModelRegistry(List<CapabilityInferrer> inferrers) {
        this.inferrers = inferrers;
    }

    /** 注册一个模型。由 ProviderFactory 在启动时调用。 */
    void register(LanguageModel model) {
        models.put(model.modelId(), model);
        log.info("模型注册: {}", model.modelId());
    }

    /** 按 ID 获取模型。 */
    LanguageModel get(String modelId) {
        LanguageModel m = models.get(modelId);
        if (m == null) throw new ModelNotFoundException(modelId);
        return m;
    }

    /** 按能力过滤可用的模型。 */
    List<LanguageModel> findByCapability(ModelCapabilities.Capability cap) {
        return models.values().stream()
            .filter(m -> m.capabilities().supported().contains(cap))
            .toList();
    }

    /** 按多个能力 AND 过滤。 */
    List<LanguageModel> findByCapabilities(Set<ModelCapabilities.Capability> required) {
        return models.values().stream()
            .filter(m -> m.capabilities().supported().containsAll(required))
            .toList();
    }

    /** 所有已注册的模型。 */
    List<LanguageModel> all() { return List.copyOf(models.values()); }
}
```

### 4.1 ProviderFactory — 从配置创建 Adapter

```java
@Component
class ProviderFactory {

    private final ModelRegistry registry;
    private final List<CapabilityInferrer> inferrers;

    /**
     * 从 yaml 配置初始化所有 provider。
     * 应用启动时调用一次。
     */
    @PostConstruct
    void init() {
        for (var providerConfig : appConfig.models().providers()) {
            LanguageModel model = createAdapter(providerConfig);
            registry.register(model);
        }
    }

    LanguageModel createAdapter(ProviderConfig config) {
        return switch (config.type()) {
            case "openai"   -> new OpenAiAdapter(config.openai(), inferCapability(config.model(), "openai"));
            case "anthropic"-> new AnthropicAdapter(config.anthropic(), inferCapability(config.model(), "anthropic"));
            case "ollama"   -> new OllamaAdapter(config.ollama(), inferCapability(config.model(), "local"));
            case "vllm"     -> new VllmAdapter(config.vllm(), inferCapability(config.model(), "local"));
            default -> throw new UnsupportedProviderException(config.type());
        };
    }

    ModelCapabilities inferCapability(String modelName, String provider) {
        return inferrers.stream()
            .map(i -> i.infer(modelName))
            .filter(Objects::nonNull)
            .findFirst()
            .orElseGet(() -> defaultCapability(modelName, provider));
    }
}
```

## 5. ModelRouter — 任务 → 模型映射

ModelRouter 根据任务特征选择最优模型。

```java
@Component
class ModelRouter {

    private final ModelRegistry registry;
    private final RoutingConfig config;

    /** 获取默认模型。适用于不需要特别能力的一般对话。 */
    LanguageModel defaultModel() {
        return registry.get(config.default());
    }

    /** 按任务复杂度选择模型：简单任务用轻量模型，复杂任务用重量级模型。 */
    LanguageModel forTask(TaskProfile task) {
        if (task.requiresThinking()) {
            // 需要推理能力 → 优先使用支持 THINKING 的模型
            return selectBest(registry.findByCapability(THINKING), task);
        }
        if (task.requiresVision()) {
            return selectBest(registry.findByCapability(VISION), task);
        }
        if (task.isSimple()) {
            // 简单任务（问候、确认、格式化）→ 使用低成本模型
            return selectCheapest(registry.findByCapability(STREAM));
        }
        return registry.get(config.default());
    }

    /** 按调用场景选择模型。 */
    LanguageModel forIntent(Intent intent) {
        return switch (intent) {
            case CHAT    -> registry.get(config.chat() != null ? config.chat() : config.default());
            case CREATIVE -> registry.get(config.creative() != null ? config.creative() : config.default());
            case ANALYSIS -> registry.get(config.analysis() != null ? config.analysis() : config.default());
        };
    }
}

/**
 * 任务画像——由调用方在请求模型时提供。
 * 描述任务的特征，ModelRouter 据此选择模型。
 */
record TaskProfile(
    boolean requiresThinking,
    boolean requiresVision,
    boolean requiresToolUse,
    int estimatedInputTokens,     // 输入量（决定上下文窗口需求）
    boolean isSimple,             // 简单任务可用低成本模型
    String taskType               // "chat" / "code" / "analysis" / "creative"
) {
    static TaskProfile simple() {
        return new TaskProfile(false, false, false, 500, true, "chat");
    }
    static TaskProfile complex() {
        return new TaskProfile(true, false, true, 8000, false, "analysis");
    }
}
```

### 5.1 回退策略

当一个模型不可用时（网络错误、限流、超时），自动回退到替代模型。

```java
class ModelRouter {
    // ...

    /**
     * 带回退的模型调用。
     * 主模型失败后自动切换到回退模型，对调用方透明。
     */
    Flux<ChatEvent> chatWithFallback(List<Msg> messages, ModelParams params, TaskProfile task) {
        LanguageModel primary = forTask(task);
        LanguageModel fallback = registry.get(config.fallback());

        return primary.chat(messages, params)
            .onErrorResume(throwable -> {
                log.warn("主模型 {} 调用失败，回退到 {}: {}",
                    primary.modelId(), fallback.modelId(), throwable.getMessage());
                return fallback.chat(messages, params);
            });
    }
}
```

## 6. 成本控制

模型选择不可忽略成本。`CostTracker` 按账户/模型维度追踪 token 消耗，并持久化到 DB。

```java
@Component
class CostTracker {

    private final MeterRegistry registry;
    private final TokenUsageRepository tokenRepo;

    void record(LanguageModel model, TokenUsage usage, UUID accountId, UUID conversationId) {
        // Prometheus 指标
        Counter.builder("llm.tokens.total")
            .tag("model", model.modelId())
            .tag("account", accountId.toString())
            .register(registry)
            .increment(usage.totalTokens());

        double cost = usage.inputTokens() * model.capabilities().inputPricePer1k() / 1000
                    + usage.outputTokens() * model.capabilities().outputPricePer1k() / 1000;
        Counter.builder("llm.cost.usd")
            .tag("model", model.modelId())
            .tag("account", accountId.toString())
            .register(registry)
            .increment(cost);

        // 持久化到 DB（对应 v1.0 TokenUsageJpaRepository）
        tokenRepo.save(new TokenUsageRecord(
            accountId, conversationId, model.modelId(),
            usage.inputTokens(), usage.outputTokens(), usage.thinkingTokens(),
            usage.totalTokens(), cost, Instant.now()
        ));
    }
}
```

## 8. 装饰器链

模型的横切关注点通过 Decorator 组合，不侵入核心 Adapter。

```java
// ———— 基础 ————
LanguageModel base = new OpenAiAdapter(config);

// ———— 重试 ————
LanguageModel withRetry = new RetryDecorator(base, 3);

// ———— 指标采集 ————
LanguageModel withMetrics = new MetricsDecorator(withRetry, registry);

// ———— 成本追踪 ————
LanguageModel withCostTracking = new CostTrackingDecorator(withMetrics, costTracker);

// ———— 注册到 Registry ————
registry.register(withCostTracking);
```

内置 Decorator：

| Decorator | 作用 | 可配置 |
|---|---|---|
| `RetryDecorator` | 网络错误/限流时自动重试 | 重试次数、退避策略 |
| `MetricsDecorator` | 采集 TTFB、TPS、P50/P95 耗时 | Micrometer Tag |
| `CostTrackingDecorator` | 记录 token 消耗和费用 | 账户维度 |
| `RateLimitDecorator` | 按模型/账户限流 | RPM / TPM |
| `FallbackDecorator` | 主模型失败切备用 | 备用模型 ID |

```java
class RetryDecorator implements LanguageModel {
    private final LanguageModel delegate;
    private final int maxRetries;

    @Override
    public Flux<ChatEvent> chat(List<Msg> messages, ModelParams params) {
        return Flux.defer(() -> delegate.chat(messages, params))
            .retryWhen(Retry.backoff(maxRetries, Duration.ofSeconds(1))
                .filter(throwable -> throwable instanceof RetryableException));
    }
}
```

## 9. 与森林架构的集成

`LeafExecutor`（特别是 `AgentLeafExecutor`）通过 `ModelRouter` 获取语言模型。

```java
class AgentLeafExecutor implements LeafExecutor {

    private final ModelRouter router;

    @Override
    public Flux<OrchestrationEvent> execute(ForestNode node, ExecutionContext ctx, EventSink sink) {
        TaskProfile task = TaskProfile.from(node);

        // ModelRouter 根据任务特征选择模型
        LanguageModel model = router.forTask(task);

        // 带回退的调用
        return router.chatWithFallback(buildMessages(node), buildParams(node), task)
            .doOnNext(event -> {
                if (event instanceof ChatEvent.Chunk chunk) {
                    sink.emit(ForestEvent.detail(node.nodeId(), chunk.text()));
                }
            })
            .ofType(ChatEvent.Completed.class)
            .singleOrEmpty()
            .flatMap(completed -> buildResult(completed.result(), node, sink));
    }
}
```

## 11. 多模态能力扩展

### 10.1 问题

`LanguageModel.chat()` 只覆盖了文本对话。图像生成、语音合成、语音识别、视频生成是不同能力——它们有不同的输入输出格式、不同的参数、不同的调用方式。但管理方式（注册、路由、成本追踪、重试）和文本模型一致。

**目标**：新增一种能力类型时，不修改任何现有代码。

### 10.2 能力接口族

每个能力有独立的接口，但共享 `Model` 容器和 `ModelRegistry`。

```java
// ———— 底层容器：一个 Provider 可以持有多个能力 ————

/**
 * 模型——一个 provider 实例持有的所有能力集合。
 * 例如同一个 OpenAI 账号可以同时支持 Chat + ImageGen + SpeechSynth。
 */
interface Model {
    /** 模型标识。与 provider 配置中的 model 字段对应。 */
    String modelId();

    /** 注册在同一个 Model 下的所有能力。 */
    List<Capability> capabilities();

    /** 按能力类型查找。返回 null 表示本模型不支持该能力。 */
    default <T extends Capability> T as(Class<T> type) {
        return (T) capabilities().stream()
            .filter(c -> type.isAssignableFrom(c.getClass()))
            .findFirst()
            .orElse(null);
    }
}

// ———— 能力标记接口 ————

interface Capability {
    /** 能力所属模型 ID。 */
    String modelId();
}

// ———— 具体能力接口（每个都是 Capability 的子类型） ————

/**
 * 文本对话能力（原有 LanguageModel 的能力）。
 * 这是唯一一个不通过 as() 获取的能力——为了向后兼容，
 * ModelRouter.defaultModel() 直接返回 ChatCapability。
 */
interface ChatCapability extends Capability {
    Flux<ChatEvent> chat(List<Msg> messages, ModelParams params);

    default Mono<ChatResult> complete(List<Msg> messages, ModelParams params) {
        return chat(messages, params)
            .ofType(ChatEvent.Completed.class)
            .singleOrEmpty()
            .map(ChatEvent.Completed::result);
    }
}

/**
 * 图像生成能力。输入文本 prompt，输出图像。
 */
interface ImageGenCapability extends Capability {
    Flux<ImageGenEvent> generate(ImagePrompt prompt, ModelParams params);
}

/**
 * 语音合成（TTS）能力。输入文本，输出音频。
 */
interface SpeechSynthCapability extends Capability {
    Flux<SpeechSynthEvent> synthesize(SpeechSynthRequest request, ModelParams params);
}

/**
 * 语音识别（STT）能力。输入音频，输出文本。
 */
interface SpeechRecogCapability extends Capability {
    Flux<SpeechRecogEvent> recognize(SpeechRecogRequest request, ModelParams params);
}

/**
 * 视频生成能力。输入文本 prompt，输出视频。
 */
interface VideoGenCapability extends Capability {
    Flux<VideoGenEvent> generate(VideoPrompt prompt, ModelParams params);
}
```

### 10.3 能力事件协议

每种能力有自己的事件类型。结构一致——使用 `sealed interface` 枚举所有可能的事件变体。

```java
// ———— 图像生成事件 ————

sealed interface ImageGenEvent {
    /** 生成进度（百分比）。某些模型逐步输出时可用。 */
    record Progress(int percent) implements ImageGenEvent {}
    /** 图像已生成，返回 URL 或 Base64。 */
    record Completed(String url, String format, int width, int height) implements ImageGenEvent {}
    /** 生成异常。 */
    record Error(Throwable cause) implements ImageGenEvent {}
}

record ImagePrompt(String prompt, String negativePrompt, String size, int n, String style) {
    static ImagePrompt of(String prompt) {
        return new ImagePrompt(prompt, null, "1024x1024", 1, null);
    }
}

// ———— 语音合成事件 ————

sealed interface SpeechSynthEvent {
    /** 音频块（流式 TTS 时逐步输出）。 */
    record AudioChunk(byte[] audio, String format) implements SpeechSynthEvent {}
    /** 合成完成。 */
    record Completed(long durationMs) implements SpeechSynthEvent {}
    /** 合成异常。 */
    record Error(Throwable cause) implements SpeechSynthEvent {}
}

record SpeechSynthRequest(String text, String voice, double speed) {
    static SpeechSynthRequest of(String text) {
        return new SpeechSynthRequest(text, "default", 1.0);
    }
}

// ———— 语音识别事件 ————

sealed interface SpeechRecogEvent {
    /** 识别文本块（流式识别时逐步输出）。 */
    record Chunk(String text, boolean isFinal) implements SpeechRecogEvent {}
    /** 识别完成。 */
    record Completed(String fullText, String language) implements SpeechRecogEvent {}
    /** 识别异常。 */
    record Error(Throwable cause) implements SpeechRecogEvent {}
}

record SpeechRecogRequest(byte[] audio, String format, String language) {}

// ———— 视频生成事件 ————

sealed interface VideoGenEvent {
    record Progress(int percent) implements VideoGenEvent {}
    record Completed(String url, String format, int durationSec) implements VideoGenEvent {}
    record Error(Throwable cause) implements VideoGenEvent {}
}

record VideoPrompt(String prompt, String size, int durationSec) {
    static VideoPrompt of(String prompt) {
        return new VideoPrompt(prompt, "1920x1080", 10);
    }
}
```

### 10.4 Provider Adapter 的多能力注册

一个 Provider Adapter 可以实现多个能力接口。

```java
// ———— OpenAI Adapter 同时提供 Chat + ImageGen + SpeechSynth ————

class OpenAiModel implements Model, ChatCapability, ImageGenCapability, SpeechSynthCapability {

    private final String modelId;
    private final WebClient client;
    private final String apiKey;

    OpenAiModel(OpenAiConfig config) {
        this.modelId = config.model();
        this.client = WebClient.create(config.baseUrl());
        this.apiKey = config.apiKey();
    }

    // == Model 接口 ==

    @Override public String modelId() { return modelId; }

    @Override
    public List<Capability> capabilities() {
        return List.of(this);  // this 实现了 ChatCapability + ImageGenCapability + SpeechSynthCapability
    }

    // == ChatCapability ==

    @Override
    public Flux<ChatEvent> chat(List<Msg> messages, ModelParams params) {
        // ... 原有的 chat 实现 ...
    }

    // == ImageGenCapability ==

    @Override
    public Flux<ImageGenEvent> generate(ImagePrompt prompt, ModelParams params) {
        return client.post()
            .uri("/v1/images/generations")
            .header("Authorization", "Bearer " + apiKey)
            .bodyValue(Map.of(
                "model", modelId,
                "prompt", prompt.prompt(),
                "n", prompt.n(),
                "size", prompt.size()
            ))
            .retrieve()
            .bodyToFlux(OpenAiImageEvent.class)
            .map(this::toImageGenEvent);
    }

    private ImageGenEvent toImageGenEvent(OpenAiImageEvent e) {
        return new ImageGenEvent.Completed(e.url(), "png", 1024, 1024);
    }

    // == SpeechSynthCapability ==

    @Override
    public Flux<SpeechSynthEvent> synthesize(SpeechSynthRequest request, ModelParams params) {
        return client.post()
            .uri("/v1/audio/speech")
            .header("Authorization", "Bearer " + apiKey)
            .bodyValue(Map.of(
                "model", "tts-1",
                "input", request.text(),
                "voice", request.voice(),
                "response_format", "mp3"
            ))
            .retrieve()
            .bodyToFlux(byte[].class)
            .map(bytes -> new SpeechSynthEvent.AudioChunk(bytes, "mp3"));
    }
}
```

### 10.5 ModelRegistry 的能力类型查询

```java
@Component
class ModelRegistry {
    private final Map<String, Model> models = new ConcurrentHashMap<>();

    void register(Model model) {
        models.put(model.modelId(), model);
    }

    // 原有按 modelId 查询（用于 chat）
    Model get(String modelId) { return models.get(modelId); }

    // 原有按 Capability 枚举过滤（用于 chat）
    List<Model> findByCapability(ModelCapabilities.Capability cap) { ... }

    /**
     * 按能力类型查询——这是多模态扩展的核心方法。
     * 返回所有持有指定能力类型的 Model。
     */
    <T extends Capability> List<T> findByCapabilityType(Class<T> type) {
        return models.values().stream()
            .map(model -> model.as(type))
            .filter(Objects::nonNull)
            .toList();
    }

    /**
     * 按能力类型查询单个——要求有且只有一个匹配。
     * 适用于"找个能 TTS 的模型"这种场景。
     */
    <T extends Capability> T findSingleByCapabilityType(Class<T> type) {
        List<T> found = findByCapabilityType(type);
        if (found.isEmpty()) throw new CapabilityNotFoundException(type);
        return found.get(0); // 多个匹配时返回第一个
    }
}
```

### 10.6 多模态路由

```java
@Component
class MultimodalRouter {

    private final ModelRegistry registry;

    /** 找个能生成图像的模型。不限定具体模型，由路由策略选择。 */
    ImageGenCapability imageGen() {
        return router.forCapability(ImageGenCapability.class);
    }

    /** 找个能 TTS 的模型。优先使用专用 TTS 模型（如 ElevenLabs），
     *  没有专用模型时回退到 Chat 模型的 TTS 子能力。 */
    SpeechSynthCapability speechSynth() {
        List<SpeechSynthCapability> dedicated = registry.findByCapabilityType(SpeechSynthCapability.class);
        if (!dedicated.isEmpty()) return dedicated.get(0);

        // 回退：Chat 模型可能没有 SpeechSynthCapability
        // 可通过 Text → LLM 生成 SSML → 交给系统 TTS
        throw new CapabilityNotFoundException(SpeechSynthCapability.class);
    }

    /** 按能力类型的路由封装——带回退和重试。 */
    <T extends Capability> T forCapability(Class<T> type) {
        T capability = registry.findSingleByCapabilityType(type);
        if (capability == null) {
            // 尝试回退路由
            return fallbackFor(type);
        }
        return capability;
    }
}
```

### 10.7 与森林架构的集成——新的叶子类型

多模态能力通过新的叶子类型接入 Forest 执行树，不涉及现有代码修改。

```java
// ———— 图像生成叶子 ————

class ImageGenTaskNode extends ExecutableNode {
    String imagePrompt;
    String size;
}

@Component
class ImageGenLeafExecutor implements LeafExecutor {
    private final MultimodalRouter router;

    @Override
    public String supportedType() { return "image_gen"; }

    @Override
    public Flux<OrchestrationEvent> execute(ForestNode node, ExecutionContext ctx, EventSink sink) {
        ImageGenTaskNode task = (ImageGenTaskNode) node;
        ImageGenCapability gen = router.imageGen();

        return gen.generate(ImagePrompt.of(task.imagePrompt), ModelParams.defaults())
            .map(event -> switch (event) {
                case ImageGenEvent.Completed c ->
                    OrchestrationEvent.complete(Map.of(
                        "type", "image",
                        "url", c.url(),
                        "format", c.format()
                    ));
                case ImageGenEvent.Error e ->
                    OrchestrationEvent.error("图像生成失败: " + e.cause().getMessage());
                default -> OrchestrationEvent.stream("生成中...");
            });
    }
}

// ———— 语音合成叶子 ————

@Component
class SpeechSynthLeafExecutor implements LeafExecutor {
    private final MultimodalRouter router;

    @Override
    public String supportedType() { return "speech_synth"; }

    @Override
    public Flux<OrchestrationEvent> execute(ForestNode node, ExecutionContext ctx, EventSink sink) {
        SpeechSynthCapability tts = router.speechSynth();
        String text = node.content();

        return tts.synthesize(SpeechSynthRequest.of(text), ModelParams.defaults())
            .map(event -> switch (event) {
                case SpeechSynthEvent.AudioChunk a ->
                    OrchestrationEvent.stream("[音频 " + a.format() + " " + a.audio().length + " 字节]");
                case SpeechSynthEvent.Completed c ->
                    OrchestrationEvent.complete(Map.of("type", "audio", "durationMs", c.durationMs()));
                case SpeechSynthEvent.Error e ->
                    OrchestrationEvent.error("语音合成失败: " + e.cause().getMessage());
                default -> OrchestrationEvent.stream("");
            });
    }
}
```

### 10.8 新增一个能力类型的工作量

| 步骤 | 动作 | 需要修改的现有代码 |
|---|---|---|
| 1 | 定义能力接口（如 `VideoGenCapability`） | 0 |
| 2 | 定义事件 sealed interface（如 `VideoGenEvent`） | 0 |
| 3 | 在某个 Model 中实现该接口 | 0（新增方法） |
| 4 | 注册 `VideoGenLeafExecutor` | 0（新增类） |
| 5 | 在 `MultimodalRouter` 中添加便捷方法 `videoGen()` | 0（新增方法） |

**零修改现有代码。** 新增能力类型只需要新增接口 + 实现 + 叶子执行器。
