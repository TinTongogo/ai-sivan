# 第八章：记忆系统

## 核心问题

大语言模型本身是无状态的。每次对话都是"失忆"的。如何让系统"记住"用户的历史、偏好、知识，并在适当的时候主动"想起"？

## 8.1 三层递进记忆

### 8.1.1 SESSION（会话级）
- 生命周期：5 小时
- 存储范围：当前对话的上下文片段
- 作用：短期记忆，保持当前对话的连贯性

### 8.1.2 PROJECT（项目级）
- 生命周期：7 天
- 存储范围：当前项目的关键信息
- 作用：中期记忆，跨对话保持项目上下文

### 8.1.3 USER（用户级）
- 生命周期：90 天
- 存储范围：用户的长期偏好、知识结构
- 作用：长期记忆，构建用户画像

### 8.1.4 自动晋升机制

`ForgettingCurveService` 每小时扫描一次，按规则晋升：

```
SESSION（访问 > 5 次，保留率 > 0.5）→ PROJECT
PROJECT（访问 > 15 次，保留率 > 0.7）→ USER
```

## 8.2 艾宾浩斯遗忘曲线

### 8.2.1 计算公式

```java
retention = exp(-λ × Δt / √accessCount)
```

其中：
- λ = 衰减因子（SESSION: 0.5, PROJECT: 0.05, USER: 0.01）
- Δt = 距上次访问的时间差
- accessCount = 访问次数（高频访问衰减更慢）

### 8.2.2 归档阈值
- 保留率 < ARCHIVE_THRESHOLD（默认 0.2）→ 自动归档
- 归档后不再参与记忆注入

## 8.3 情境闪现（Flashback）

### 8.3.1 触发时机
- 每次用户输入后
- `FlashbackScanner.scan()` 在上下文构建前被调用

### 8.3.2 候选评分

```java
score = importanceWeight × levelWeight × forgettingFactor × frequencyFactor + contextBonus
```

- `importanceWeight`：重要标记 ×2.5
- `levelWeight`：SESSION=0.3, PROJECT=0.8, USER=1.0
- `forgettingFactor`：1.0 - retention（即将遗忘的更有价值）
- `frequencyFactor`：log(accessCount + 1) + 0.5
- `contextBonus`：关键词匹配

### 8.3.3 注入流程

```
FlashbackScanner.scan()
  → 按层级检索所有未归档记忆
    → 计算每个记忆的 relevanceScore
      → 按分数排序，取 Top-K
        → 注入 MemoryTree → 成为当前对话的上下文
```

## 8.4 记忆的存储演进

### 8.4.1 第一阶段：独立记忆表
- 独立的 `memory_entries` 表
- 独立的仓储接口和实现
- 问题：与森林架构脱节，无法利用树结构

### 8.4.2 第二阶段：合入 forest_nodes
- V25 迁移：`memory_entries` → `forest_nodes(node_type='memory')`
- 所有记忆使用 `metadata` JSONB 存储扩展字段
- 问题：`level`/`archived`/`important` 在 JSONB 中，无法走索引

### 8.4.3 第三阶段：加专用列（当前）
- V30 迁移：`level`/`archived`/`important`/`scope_id`/`summary` 作为专用列
- 部分索引覆盖所有查询模式
- SQL WHERE 过滤替代 JSONB 内存过滤

### 8.4.4 MemoryEntry 与 MemoryNode 的双重表示
- `MemoryEntry`（`sivan-domain/memory/`）：领域实体，用于 IMemoryRepository 的 CRUD
- `MemoryNode`（`sivan-domain/forest/tree/node/`）：树节点类型，用于 MemoryFoldStrategy 的折叠判断
- 两者不合并：职责不同，前者是数据模型，后者是树形模型

## 8.5 与相邻模块的交互

```
对话系统 → FlashbackScanner（记忆召回）
  → MemoryTree 构建 → ContextForest 构建
    → LLM 调用（记忆作为上下文注入）

定时任务 → ForgettingCurveService（遗忘曲线计算）
  → MemoryRepositoryAdapter.update()（更新保留率）

执行树压缩 → MemoryFoldStrategy（折叠低重要性记忆）
  → ForestCompressor（预算分配）
```
