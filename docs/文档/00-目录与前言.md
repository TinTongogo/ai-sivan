# 灵枢（Sivan）— 私人 AI 团队操作系统

## 副标题：从零构建一个多智能体协作平台

---

### 前言

2024 年，大语言模型的能力已经足够强大，但单次对话的线性交互模式严重限制了它的潜力。每一次对话都是一次"失忆"——之前的推理链路、中间产物、决策记录全部消失。Sivan（灵枢）项目正是在这个背景下诞生的：**将单次对话扩展为跨轮次的项目推进，将单 Agent 回复升级为多角色团队的协作流水线**。

本书不是一份 API 文档，也不是一份部署指南。它是一份深入的架构解析，按**功能模块**而非代码模块组织，着重讲解每个模块解决了什么问题、为什么这样设计、以及与相邻模块的交互细节。

---

### 读者对象

- 对多智能体系统架构感兴趣的后端开发者
- 正在构建 AI 原生应用的架构师
- 想理解"森林架构"（Forest Architecture）设计思想的技术读者

### 前置知识

- Java 21、Spring Boot 3、WebFlux 基础
- PostgreSQL + pgvector 基础
- 对 LLM 调用模式（ReAct、Function Calling）有基本了解

---

## 全书结构

全书分为 **4 个部分、14 章**，按功能模块递进组织：

### 第一部分：基石

| 章 | 标题 | 核心内容 |
|----|------|---------|
| 1 | **对话系统** | 消息模型、流式 SSE、多轮上下文、重排与编辑 |
| 2 | **森林架构** | ContextForest、ContextTree、Epoch、TopicNode 的设计哲学 |
| 3 | **消息与执行树的关联** | 消息节点、Assistant 消息的执行树链接、回退策略 |

### 第二部分：智能体系统

| 章 | 标题 | 核心内容 |
|----|------|---------|
| 4 | **组合式路由系统** | 三层贝叶斯路由（精确→语义→Thompson采样→LLM创建）、技能独立匹配管道 |
| 5 | **编排策略** | SEQUENTIAL/PARALLEL/CONDITIONAL/HIERARCHICAL/CONSENSUS 五种模式的设计与实现 |
| 6 | **Agent 执行器** | ReAct 循环、工具调用、Agent-to-Agent 通信、HITL 人机协同 |
| 7 | **叶子节点体系** | Chat/ImageGen/VideoGen/SpeechRecog/SpeechSynth 等各类 LeafExecutor |

### 第三部分：持久化与上下文

| 章 | 标题 | 核心内容 |
|----|------|---------|
| 8 | **记忆系统** | 三层递进记忆（SESSION→PROJECT→USER）、艾宾浩斯遗忘曲线、情境闪现 |
| 9 | **上下文压缩** | 微压缩/自动摘要/向量检索三层管道、Token 预算分配 |
| 10 | **工具系统** | 双轨工具（内部 Bash/File + 外部 MCP）、ToolRouter、工具注册与安全沙箱 |

### 第四部分：运维与演进

| 章 | 标题 | 核心内容 |
|----|------|---------|
| 11 | **持久化与恢复** | 数据库设计（forest_nodes 统一表）、递归 CTE 树加载、断点恢复 |
| 12 | **可观测性** | 执行链路追踪（Span/Trace）、指标采集、SSE 进度推送 |
| 13 | **安全模型** | JWT 认证、三层沙箱、审计日志、速率限制 |
| 14 | **架构演进实录** | 从 V1 目标管理系统到 V3 森林架构的演进、metadata 治理、拆表与合表的决策 |
